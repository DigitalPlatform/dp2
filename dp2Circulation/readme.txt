

这是 v2 分支。采用 .NET Framework 4.0 编译


===

修改一个分馆的馆代码，需要做如下一些操作:

1) 将读者库定义中，符合这个馆代码的做出修改。
这里要注意的是，将来可能会允许一个读者库定义为 海淀分馆/二级分馆 的馆代码。若修改一级馆代码“海淀分馆”，也别忘了修改这样的字符串

2) 系统管理窗的“馆藏地”属性页，要把相应的馆藏地的馆代码修改

3) 系统管理窗的“排架体系”属性页，要把相应的馆藏地的馆代码修改

4) 系统管理窗的“脚本程序”，只能手工修改了

5) 值列表可能需要修改

6) 流通权限中的馆代码需要修改

7) 册记录中，符合的馆代码字段内容要修改。这个需要单独进行批处理了，时间较长
而且需要注意在 library.xml 中定义了 SetEntities() API 保存册记录时强制校验馆代码的效果对这种批处理的影响
